# Contributing to PlugZone

Thank you for your interest in contributing to PlugZone! We welcome contributions from everyone who shares our passion for privacy and freedom.

## How to Contribute

1. **Fork the Repository**: Create your own fork of the project.
2. **Create a Branch**: Create a new branch for your feature or bug fix.
   ```bash
   git checkout -b feature/amazing-feature
   ```
3. **Commit Your Changes**: Make your changes and commit them with clear, descriptive messages.
   ```bash
   git commit -m "Add amazing feature"
   ```
4. **Push to the Branch**: Push your changes to your fork.
   ```bash
   git push origin feature/amazing-feature
   ```
5. **Open a Pull Request**: Submit a pull request to the main repository.

## Coding Standards

- **PHP**: We follow PSR-12 coding standards.
- **Laravel**: Follow standard Laravel best practices.
- **Security**: Security is paramount. Always sanitize inputs, escape outputs, and validate data.
- **No JavaScript**: This project aims to function without client-side JavaScript. Please respect this constraint unless absolutely necessary and discussed.

## Security Vulnerabilities

If you discover a security vulnerability within PlugZone, please do **NOT** open a public issue. Instead, please contact the maintainers privately or look for a security policy in the repository.
